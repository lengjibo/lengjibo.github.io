VSFTP

centos下ftp服务器搭建

开搞开搞

安装并启动ftp服务

使用yum安装ftp

yum install vsftpd -y

安装完成后，启动 FTP 服务：

service vsftpd start

查看是否已经监听21端口

netstat -nltp | grep 21

注：-nltp的作用是显看已连接的TCP端口，以及PID，并且搜索端口号为21的服务

此时访问ftp://服务器地址/ 便可以浏览我们的/var/ftp目录下的内容了

目前 FTP 服务登陆允许匿名登陆，也无法区分用户访问，我们需要配置 FTP 访问权限

vsftpd 的配置目录为 /etc/vsftpd，包含下列的配置文件：

vsftpd.conf 为主要配置文件

ftpusers 配置禁止访问 FTP 服务器的用户列表

user_list 配置用户访问控制

配置ftp权限

限制匿名登录与切换根目录

编辑 /etc/vsftpd/vsftpd.conf，找到下面两处配置
并修改


 禁用匿名用户
anonymous_enable=NO

禁止切换根目录
chroot_local_user=YES

修改完成后重启ftp服务即可

新建ftp用户并设置权限

useradd ftpuser

设置密码：

echo “V9vYqae7” | passwd ftpuser –stdin

stdin为在标准输入中读取密码

限制用户 ftpuser 只能通过 FTP 访问服务器，而不能直接登录服务器：

usermod -s /sbin/nologin ftpuser

为用户设置目录

新建个目录

mkdir -p /data/ftp/pub

设置访问权限：

chmod a-w /data/ftp && chmod 777 -R /data/ftp/pub

a-w表示取消写入权限

设置为用户的主目录：

usermod -d /data/ftp ftpuser

登录

此时我们已经完成了对ftp的设置，我们现在访问ftp服务器会弹出让我们登录的提示框

！

ubuntu下ftp服务器搭建

安装 VSFTPD

sudo apt-get install vsftpd -y

其余的基本一样了，在此不再赘述

ftp配置

sudo chmod a+w /etc/vsftpd.conf

然后修改配置


 限制用户对主目录以外目录访问
chroot_local_user=YES

指定一个 userlist 存放允许访问 ftp 的用户列表
userlist_deny=NO
userlist_enable=YES

记录允许访问 ftp 用户列表
userlist_file=/etc/vsftpd.user_list

不配置可能导致莫名的530问题
seccomp_sandbox=NO

允许文件上传
write_enable=YES

使用utf8编码
utf8_filesystem=YES

然后

新建文件 /etc/vsftpd.user_list，用于存放允许访问 ftp 的用户

最后重启ftp服务

sudo systemctl restart vsftpd.service

Docker搭建ftp

执行

docker search vsftp

挑选星际高的，pull下来

运行

docker run 镜像名称

完成
